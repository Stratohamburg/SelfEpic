# 游戏大厅菜单架构设计文档 (Game Lobby Menu Architecture)

## 1. 概述
本文档旨在详细说明游戏大厅菜单的全新架构设计。新架构将大厅分为三个主要功能模块，通过底部导航栏进行切换，以提升玩家的操作体验、明确系统功能划分。此文档作为《核心玩法GDD》的补充，专注于大厅系统、养成系统与抽奖系统的设计。

## 2. 全局UI与导航
- **全局常驻信息栏 (Top Bar)**：位于界面最上方，在任何Tab下均保持可见。
  - **玩家信息**：玩家头像、昵称、当前等级及经验条。
  - **核心资源**：金币数量、钻石数量、抽奖券数量（点击资源图标可弹出获取途径或快捷购买提示）。
- **底部导航栏 (Bottom Tab Bar)**：包含三个固定标签页，从左到右依次为：
  1. **商店 (Shop)**
  2. **冒险 (Adventure)**
  3. **装备 (Equipment)**
- **默认界面**：玩家进入游戏大厅或每次登录游戏时，默认选中并展示**冒险 (Adventure)**界面。

## 3. 核心界面详细设计

### 3.1 冒险界面 (Adventure) - 默认Tab
- **定位**：游戏的核心战斗入口。
- **视觉与氛围**：
  - **背景**：展示当前关卡的动态游戏世界背景，或主角的待机动画（Idle Animation），增加沉浸感。
  - **进度展示**：界面中上部区域展示当前关卡的名称、章节进度（如“第3章-幽暗密林 3/10”）。
- **核心交互**：
  - **出发按钮 (GO!)**：位于界面中间偏下的位置。视觉上作为当前界面的绝对焦点，使用醒目的颜色、呼吸灯动效或粒子特效。
  - **交互逻辑**：点击“GO!”按钮后，直接触发战斗逻辑，加载并进入冒险场景。

### 3.2 商店界面 (Shop) - 抽奖与资源获取
- **定位**：游戏的抽奖（Gacha）与资源获取中心。
- **核心元素**：
  - **抽奖操作区**：提供“单抽”和“十连抽”按钮，按钮上明确显示消耗的货币类型及数量（如消耗钻石或抽奖券）。
  - **奖池内容**：玩家通过抽奖可获得以下类型的道具：
    - 武器 (Weapons)
    - 宠物 (Pets)
    - 武器升级材料 (Weapon Upgrade Materials)
    - 宠物升级材料 (Pet Upgrade Materials)
    - 武器碎片 (Weapon Fragments)
    - 宠物碎片 (Pet Fragments)
- **概率配置与公示**：
  - **底层支持**：抽奖概率必须支持通过配置文件（如 JSON/Excel）或服务器后台进行动态配置。
  - **UI展示**：界面上需提供“概率公示”或“详情”按钮，点击后弹出面板，向玩家透明展示各类道具及不同稀有度的获取概率。
- **扩展机制（推荐方向）**：
  - **抽奖演出**：加入开箱或召唤的动画效果，提升抽奖期待感。
  - **保底机制**：增加保底UI提示（如“再抽X次必得稀有武器/宠物”），提升玩家付费/抽奖意愿。

### 3.3 装备界面 (Equipment) - 养成与策略搭配
- **定位**：玩家的养成、策略搭配与资产管理中心。
- **布局结构**（界面从上到下划分为三个主要区域）：

  #### 第一排：出战序列设置区 (Attack Order)
  - **功能**：玩家在此设置武器和宠物的攻击/行动顺序。
  - **展示方式**：横向排列的装备槽位。槽位之间使用明显的**箭头符号**（如 `[槽位1] ➔ [槽位2] ➔ [槽位3]`）连接，直观表达先后顺序。
  - **交互**：支持玩家将下方列表中的武器/宠物拖拽至此，或通过点击下方列表选中后点击“装备”按钮上阵。支持拖拽调整已上阵物品的顺序。

  #### 第二排：资产列表区 (Inventory)
  - **功能**：展示玩家当前拥有的所有武器和宠物。
  - **展示方式**：以网格列表或横向滑动列表的形式展示。
  - **视觉**：使用**小图标**展示，图标边框颜色或角标反映物品的稀有度、等级或星级。
  - **分类与筛选**：在此区域顶部增加“全部”、“武器”、“宠物”的分类页签，方便玩家快速查找。

  #### 第三排：详情展示区 (Details)
  - **功能**：展示选中物品的详细信息及养成操作。
  - **交互**：当玩家点击第二排的某个小图标时，该区域会动态刷新为对应物品的详情。
  - **展示内容**：
    - 物品大图标、名称、稀有度、当前等级。
    - 核心属性（如攻击力、生命值、攻速等）。
    - 技能描述或特殊被动效果。
    - **养成按钮**：如“升级”、“突破/升星”等操作按钮。按钮旁需显示消耗的对应材料/碎片及金币数量，材料不足时标红提示。

## 4. 数据与配置结构设计
- **抽奖配置表 (GachaConfig)**：
  - 字段：`ItemID`, `ItemType`, `Weight` (权重), `IsGuaranteed` (是否保底池物品)。
- **玩家出战序列 (BattleFormation)**：
  - 数据结构：`List<Entity>` 或 `Array`，严格按照索引顺序决定战斗中的出手/生效顺序。
- **物品数据模型 (ItemModel)**：
  - 包含基础属性、成长系数、技能ID、所需升级材料ID及数量公式。

## 5. 技术实现备忘录 (To-Do)

2. **事件联动机制**：装备界面中，第二排（列表）的点击事件需要绑定委托/回调（Event/Observer Pattern），实时更新第三排（详情）的UI渲染。
3. **拖拽与排序逻辑**：实现第一排槽位的拖拽排序（Drag & Drop）功能，并在排序完成后同步更新本地/服务端的出战序列数据。
